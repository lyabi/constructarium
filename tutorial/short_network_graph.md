# Inheritance Network Graph

The Inheritance Network visualises how grammatical constructions relate to each other as a directed diagram. Construction Grammar distinguishes four link types: **instance** (specific inherits from general), **subpart** (complex construction linked to its parts), **polysemy** (basic sense linked to extended senses), and **metaphorical** (constructions connected via conceptual metaphor, e.g. CHANGE IS MOTION linking caused-motion and resultative). Each type is rendered with a distinct visual style. The graph is a DAG, not a tree since a construction can have more than one parent. 

Users can click nodes to navigate to a construction's detail page, hover for a meaning tooltip, and zoom and pan as the network grows. A legend explains edge styles and node colours by cluster. Filters by cluster or relation type are a planned nice-to-have for when the dataset is larger.

The data comes from two existing JSON files (`constructarium_inheritance.json` and `constructarium_constructions.json`); no new files are needed. The graph will be implemented with **Cytoscape.js** and the **dagre** layout plugin which seems to be the best choice: plain SVG was ruled out because it requires hardcoded positions that don't scale. D3-force was ruled out because its physics-based layout produces an organic arrangement with no inherent hierarchy — unsuitable for an inheritance network where depth and direction carry meaning. Vis-network is a valid alternative but offers less DAG layout flexibility. Cytoscape with dagre computes a layered hierarchical layout from the graph topology automatically, handles multiple parents, and includes zoom, pan, and event handling out of the box.
